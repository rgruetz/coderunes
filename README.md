# coderunes

Generate a `REPO_MAP.md` at the root of any Node.js repository — a token-efficient table of contents that lists every file's public exports. Designed to be loaded into a coding agent's context (Claude Code, Cursor, etc.) so the agent can navigate by *file* rather than scanning the codebase to find symbols.

The map shows **signatures only**, not bodies — that's the whole point. Agents read the map first and only open files they actually need.

## Why

Coding agents waste a lot of tokens hunting for symbols and reading files just to understand a project's shape. A small, always-loaded map fixes that:

- **Compact** — function/class signatures only, bodies stripped, lines capped at 120 chars by default.
- **Deterministic** — same input produces byte-identical output, so it diffs cleanly and can be committed.
- **Per-project, file-system based** — generated by a CLI you can run locally, in pre-commit, or in CI. No daemon, no MCP server, no embeddings.

## Install

```bash
npm install --save-dev coderunes
npx coderunes init
npm run build:map
```

`coderunes init` drops a starter `coderunes.config.json` and adds two scripts to your `package.json`:

```json
{
  "scripts": {
    "build:map": "coderunes",
    "check:map": "coderunes --check"
  }
}
```

Commit the generated `REPO_MAP.md` so agents always have it available.

## CLI

```
coderunes                 Generate REPO_MAP.md in the current directory
coderunes --check         Exit non-zero if existing REPO_MAP.md is stale (CI mode)
coderunes --config <p>    Use a specific config file
coderunes --output <p>    Override the output path
coderunes init            Drop a starter coderunes.config.json + npm scripts
coderunes --help
coderunes --version
```

Exit codes: `0` success, `1` internal error, `2` `--check` found a stale or missing map.

## Configuration

Resolution order (first match wins):

1. `--config <path>` flag
2. `coderunes.config.js` / `.mjs` / `.cjs` / `.json` (walked up from `cwd`)
3. `package.json` `"coderunes"` key
4. Built-in defaults

```ts
interface RepoMapConfig {
  include?: string[];          // globs. Default: ['src/**/*.{ts,tsx,js,jsx,mjs,cjs}']
  ignore?: string[];           // globs. Default: node_modules, dist, build, *.test.*, *.spec.*, *.d.ts
  output?: string;             // default: 'REPO_MAP.md'
  maxSignatureLength?: number; // default: 120. Set to 0 to disable truncation.
  groupByDirectory?: boolean;  // default: false. When true, adds ## directory headers above ### file headers.
  includeFileSummary?: boolean;// default: false. Pulls top-of-file JSDoc as a one-line description.
  header?: string;             // optional custom header text prepended to the output
}
```

`.gitignore` is respected automatically (and `.coderunesignore` if present).

## Output format

```markdown
# Repo Map

_Auto-generated by coderunes. Do not edit by hand. Run `npm run build:map` to refresh._

### src/auth/session.ts
- `export interface SessionOptions { ttlSeconds: number; rolling?: boolean; }`
- `export function createSession(userId: string, opts?: SessionOptions): Session`
- `export class SessionStore`

### src/auth/middleware.ts
- `export async function requireAuth(req, next: () => Promise<void>): Promise<void>`
```

Behavior:

- Files are sorted alphabetically by path for stable diffs.
- Files with no exports are skipped.
- Function and class **bodies are cut**; types and interfaces are **kept intact** (flattened to one line).
- Re-exports (`export * from`, `export { x } from`) are preserved — they're navigationally important.
- Parse errors are logged to stderr and the file is skipped; one bad file doesn't fail the run.

## CI integration

Add a check step that fails if the committed map is out of date:

```yaml
- run: npx coderunes --check
```

Or as a pre-commit hook with Husky:

```bash
npx coderunes --check || (npx coderunes && git add REPO_MAP.md)
```

## Reference the map from your agent instructions

Tell your agent (e.g. in `AGENTS.md` or `CLAUDE.md`) to read `REPO_MAP.md` first:

```markdown
Before searching the codebase, read REPO_MAP.md — it lists every file's public exports.
Only open a file if its signatures suggest the symbol you need is there.
```

## Languages

v0.1 supports JavaScript, TypeScript, JSX, and TSX. Parsing is powered by [`@ast-grep/napi`](https://www.npmjs.com/package/@ast-grep/napi).
